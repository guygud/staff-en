# Таблицы геймдизайна (`new/`)

Здесь лежат **упрощённые CSV** для повседневной работы дизайнера. Они задают архитектуру контента (симптомы, бафы, карты, матрицы, нарратив уровней, наследственные дебафы, решения).

## Как играть (плейтест)

1. Из **корня репозитория** запустите локальный сервер (иначе браузер не отдаст CSV): `python3 -m http.server 8765`.
2. Откройте **`http://127.0.0.1:8765/playtest.html`**.
3. Выберите уровень → прочитайте бриф → **«К игре»**.
4. **Матрицы:** если есть свободные слоты доп. матриц, нажмите пустой — выберите матрицу из списка (как у карт). У установленной опциональной матрицы есть **«Сменить»** и **«Снять»** в заголовке блока её слотов.
5. **Карты:** нажмите пустой слот клетки → выберите карту. Повторный клик по заполненному слоту очищает карту.
6. Следите за **золотом** и **аурой** (и лимитом ауры после обрядов). **«Как пройти»** открывает справку из **«Решения.csv»** (цепочка `why`, матрицы, пошаговая сборка).
7. **↑** — проверить решение: победа или поражение; текст при активном симптоме берётся из **«Ответы на дебафы.csv»**.

## CSV-only плейтест

Откройте в браузере через локальный сервер (из корня репозитория):

```bash
python3 -m http.server 8765
```

Затем: `http://127.0.0.1:8765/playtest.html` — страница подгружает **10** файлов `new/*.csv` и считает исход через [`lib/simulate.mjs`](../lib/simulate.mjs). **Текст при провале** берётся **только** из ячейки «Алхимия систем - Ответы на дебафы.csv» для текущего `level_id` и имени симптома (колонка = имя из «Дебафы»), без `game-data.json`. Кнопка **«Как пройти»** на экране сборки показывает строки из «Решения.csv» (инструкция, бюджет из таблицы, матрицы, сборка по слотам).

## Сборка в `game-data.json`

Из корня репозитория:

```bash
node scripts/build-game-data.mjs
node scripts/validate-levels.mjs
```

Скрипт сборки:

- читает текущий `game-data.json` как основу (ID, wiring, `conflictPairs`, …);
- накладывает данные из `new/*.csv`;
- подтягивает недостающие статусы и восстанавливает `criticalStatuses`, `defaultInstalled`, цены матриц из `exports/csv/` (02, 07, 09, 11), чтобы не ломать машинную целостность;
- пишет `meta.compiledFrom: "new"`, `meta.compiledAt`, `meta.playtestBrief` (только поля из `new/*.csv` для **текста** брифинга и сайдбара — требования и подсказки из JSON там не показываются).

## Файлы

| CSV | Назначение |
|-----|------------|
| Алхимия систем - Дебафы.csv | Иконки и имена симптомов |
| Алхимия систем - Бафы.csv | Теги лечения → какие симптомы снимает |
| Алхимия систем - Заклинания.csv | Карты-заклинания (gold, aura, слоты, 3 ячейки) |
| Алхимия систем - Обряды.csv | Обряды + `aura_limit_delta` |
| Алхимия систем - Матрицы.csv | Пресеты матриц (слоты, описание; без переноса install на инстансы) |
| Алхимия систем - Уровни - матрицы.csv | На уровне: `max_matrix`, пресеты «всегда на поле» (`preset` + `preset_matrix`) и лимит опциональных матриц |
| Алхимия систем - Уровни - нарратив.csv | Нарратив, бюджет, тексты успеха |
| Алхимия систем - Наследственные дебафы.csv | Дополнительный union к `criticalStatuses` (долги не пересобираются отсюда) |
| Алхимия систем - Решения.csv | Справочные успешные пути: `why`, бюджет, матрицы, `assembly` — в плейтесте кнопка «Как пройти» |
| Алхимия систем - Ответы на дебафы.csv | Ответ плейтеста при активном симптоме на уровне (текст ячейки как в таблице, напр. `L1_MILLER_BARN-Путаница`) |

## Опциональные алиасы имён

Если имя в CSV не совпадает с `name` в `game-data.json`, добавьте маппинг в [`scripts/new-mappings.json`](../scripts/new-mappings.json) (поля `statusNameAliases`, `cardNameAliases`).

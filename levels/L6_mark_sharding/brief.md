# L6 — Марк: Шардинг

## Нарратив

> «Данные растут — один узел уже не справляется. Нужно consistent hashing, чтобы каждый ключ шёл в нужный шард. И если один шард упадёт, система должна продолжать работать с остальными. Я не готов к переполнению и к хрупкости.»
> — Марк, infra lead

Марк масштабирует архив: **три подсистемы** — маршрутизация к шардам, согласованность схемы/миграций и наблюдаемость. Уровень теперь учит сложным каскадам: карты-лечения могут породить сразу два новых долга. На маршрутизации остаётся урок **несовместимости политик**: два разных «закона» ключа (`S_SHARD_KEY` и `S_HASH_ROUTING`) без **Арбитра** дают **F2_CONTRADICTION**.

## Технологический смысл

Шардирование: consistent hash / ключ шардирования, hot-key split, dry-run миграций, кворум реплик, репликация как альтернатива полному CI/CD, ограда + зонд, схема + миграции + rollback, логи + фильтр (или ритуал снижения ауры).

## Условия победы (движок)

Закрыть три унаследованных долга (`D_ROUTING_AMBIGUITY`, `D_SCHEMA_DRIFT`, `D_NO_OBSERVABILITY`) и не нарушить критические статусы. Все три стартовых долга теперь ведут в новые L6-дебафы: `F11_HOTSPOT`, `F12_SPLIT_BRAIN`, `F13_DARK_SHARD`.

## Ограничения

| Параметр | Значение |
|---|---|
| Монеты | **24** |
| Аура | **5** |
| Матрицы | пять опциональных (`net_l6`, `obs_l6`, `st_l6`, `cd_l6`, `buffer_l6`), `cd_l6` теперь включает **CD_TESTING** |

## Заказчик не простит

| Статус | Почему |
|---|---|
| F1_OVERFLOW | Дисбаланс шардов, переполнение логов |
| **F2_CONTRADICTION** | `S_SHARD_KEY` + `S_HASH_ROUTING` без `S_POLICY_RESOLVER` |
| F5_FRAGILITY | Схема без миграции/CI/CD; репликация без синхронизации |
| F11_HOTSPOT | Горячий диапазон или непроверенный backfill |
| F12_SPLIT_BRAIN | Репликация без закреплённого кворума |
| F13_DARK_SHARD | Нет наблюдаемости по шардам |
| F7_CONFUSION | Ограда без зонда и т.п. |

## Карта уровня

См. `game-data.json` → `L6_MARK_SHARDING` (`availableMatrices`, `availableSpells`, **`solutions`** — два проходных пути A/B).

## Подсказки

См. `hints` в данных уровня; там выделены маршрутизация, двойные хвосты `S_REBALANCER`, dry-run миграций, кворум репликации, ловушка буфера и конфликт F2.

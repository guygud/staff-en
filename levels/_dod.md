# Definition of Done

## DoD уровня

Уровень переходит в статус `done` в `_campaign.md` только если выполнены все пункты:

- [ ] `brief.md` заполнен по структуре (нарратив, технологический смысл, DoD, ограничения, критические статусы, предустановки, матрицы, карты, урок, подсказки) — без пропусков.
- [ ] `solutions.md` содержит ≥ 2 валидных решения и ≥ 1 «обучающую ловушку»; для блока C — оба пути с разным типом компромисса (архитектурный vs процессный).
- [ ] Данные уровня в `game-data.json` используют актуальные поля (`coinsBudget`/`auraLimit`, не `budgetGold`/`riskLimit`).
- [ ] `scripts/validate-levels.mjs` зелёный (группы A и B) для этого уровня.
- [ ] Тексты briefing/success/fail/hints не раскрывают «правильную сборку» явно — проверено по чек-листу ниже.
- [ ] Ручной плейтест: уровень проходится за 1–3 итерации; причина каждого провала понятна из result-экрана без угадывания.
- [ ] Запись в `playtest.md` с датой и 3–5 строками наблюдений.

### Чек-лист: «подсказки не раскрывают ответ»

- Подсказки напоминают закономерности («проверь долги», «посмотри на рост»), а не говорят «возьми карту X».
- Success report не перечисляет конкретные карты, которые «сработали».
- Советы в fail report говорят «что поменять» (категория), а не «какую карту добавить».

## DoD кампании

MVP кампания считается готовой, когда:

- [x] Все 7 уровней в статусе `validated` (ручной плейтест и `done` — следующий шаг).
- [x] `_mechanics_coverage.md` — каждая механика из таблицы имеет хотя бы один `intro` и хотя бы один `reuse` (кроме механик, которые вводятся на последнем уровне).
- [x] На каждом уровне ≤ 1 новая концепция (проверяется по колонке «Новая концепция» в `_campaign.md`).
- [x] Все статусы (F1–F7) хотя бы раз встречаются в `fatalStatuses` хотя бы одного сценария в фикстурах.
- [x] Все обряды из `obryadCatalog` используются хотя бы в одном сценарии кампании.
- [x] Все карты из `availableSpells`/`availableObryads` уровней имеют хотя бы один совместимый слот на этом уровне (валидатор группа A).
- [x] `node scripts/validate-levels.mjs` завершается с кодом 0 — **312/312 проверок пройдено**.

> **Примечание:** F2_CONTRADICTION используется в L3 (ловушка — конфликт без арбитра). F7_CONFUSION введён в L1 через ловушку «Перо Всевидящего». Все статусы покрыты.
> Ручной плейтест по всем 7 уровням — ожидает записи в `playtest.md`.

## Команды

```bash
# Запустить валидатор
node scripts/validate-levels.mjs

# Запустить только для одного уровня
node scripts/validate-levels.mjs L1_MILLER_BARN
```

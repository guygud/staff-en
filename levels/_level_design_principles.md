# Принципы левел-дизайна

Задокументировано по итогам работы над L3 («Указатель на перепутье»).

---

## 1. Ресурсы — нет бесплатного

**Карты и ритуалы:** каждая карта обязана иметь ненулевые значения `gold` И `aura`.
- `gold ≠ 0` — карта чего-то стоит
- `aura ≠ 0` — карта что-то делает с риском (добавляет сложность или снижает)

**Матрицы:** `installCostGold ≥ 1`. Бесплатных матриц нет — установка инфраструктуры всегда что-то стоит.

**Почему:** если ресурс бесплатный, игрок ставит его «на всякий случай» и теряет смысл принятия решений.

---

## 2. Унаследованные долги — главный визуальный сигнал

Условие победы формируется через **унаследованные долги** (`defaultInstalled.addsDebts`), а не через скрытые требования (`BUFF_REQUIRED`).

Игрок видит:
- Бейджи унаследованных дебафов на экране брифинга и сайдбаре
- Критические статусы в итоговом отчёте если не устранены

**Не использовать `BUFF_REQUIRED`**: игрок не видит связи между требованием и конкретным буффом на карте. Если долг уже покрывает условие — `requirements` оставляем пустым.

---

## 3. Два пути: каскад vs чистое решение

| | Путь A (дешёвый) | Путь B (дорогой) |
|---|---|---|
| Карты | дешёвые (~2g, 1a), создают каскады | дорогие (~8g, 1a), без каскадов |
| Каскад | DNS → TLS, ROUTING → HEALTHCHECK | нет |
| Ритуал | не нужен | обязателен (без него > бюджет) |
| Карт | ≥ 4 | 2–3 |
| Аура | = базовый лимит (tight!) | < лимита, запас есть |
| Нарратив | «собрал из простого» | «дорого, зато надёжно» |

**Почему два пути:** учит тому, что инженерные решения — это trade-off между простотой и чистотой.

---

## 4. Каскадная механика

Дешёвая карта решает один долг, но создаёт новый:

```
S_DNS (2g) → закрывает D_SIGN_UNCLEAR
           ↳ добавляет D_DNS_EXPOSED → F3_COMPROMISE
             S_TLS_CERT (2g) → закрывает D_DNS_EXPOSED

S_TRAFFIC_SPLIT (2g) → закрывает D_ROUTING_AMBIGUITY
                      ↳ добавляет D_SPLIT_OVERLOAD → F5_FRAGILITY
                        S_HEALTHCHECK (2g) → закрывает D_SPLIT_OVERLOAD
```

**Посыл:** простое решение порождает новую проблему. Это нормально — надо просто предусмотреть следующий шаг.

Дорогая карта не создаёт каскада, но стоит в 4× дороже.

---

## 5. Три матрицы на выбор

Предлагать 3 матрицы, игрок устанавливает 2 (`maxOptionalMatrices: 2`):

| Тип | Назначение |
|---|---|
| «Полная» | Содержит все нужные слоты для любого пути |
| «Экономная» | Содержит меньше слотов, но достаточно для основных путей. Дешевле |
| «Ловушка» | Выглядит разумно, но в сочетании с одной из других — блокирует ключевой слот (TLS, DISCOVERY и т.п.) |

**Пример L3:** infra (DNS+TLS+DISC) + routing (ROUTING×3) = полная; relay (DNS+ROUTING) = экономная; relay+routing без TLS = ловушка.

Несколько инстансов одного `matrixId` → обязательно задать `name` и `description` на уровне инстанса, иначе в UI все будут выглядеть одинаково.

## 6. Бюджет

```
budget.gold  = стоимость дешёвого пути (карты + матрицы)
budget.aura  = суммарная аура дешёвого пути (tight = ровно в лимит)

Без ритуала дорогой путь > budget.gold  →  провал
С ритуалом дорогой путь ≤ budget.gold   →  проходит
```

Разрыв цен: дешёвые карты в 4× дешевле дорогих. Это делает разницу ощутимой.

---

## 7. Обязательные артефакты уровня

На каждый уровень:

1. **`game-data.json`** — `availableSpells`, `availableObryads`, `availableMatrices` (с уникальными именами), `budget`, `solutions`, `hints`
2. **`scripts/level-fixtures/LEVEL_ID.expected.json`** — минимум:
   - 2 passing сценария (оба пути)
   - 3+ trap сценария (без ритуала, каскад без закрытия, ловушечная комбинация матриц)
3. **`levels/LN_name/solutions.md`** — оба пути с таблицей карт/стоимостей, ловушки, объяснение ценового разрыва
4. **Валидация**: `node scripts/validate-levels.mjs` должен проходить без ошибок по данному уровню

---

## 8. Ритуалы

- Ритуалы не решают долги — они модифицируют ресурсы (бюджет/аура)
- Ритуал с `gold < 0` даёт скидку (делает дорогой путь возможным) → триггерит **Финал B**
- Ритуал с `auraLimitDelta > 0` расширяет лимит ауры
- Может быть несколько ритуалов с разным risk-profile (R_PATRON_SEAL = безопасный, R_AI_BUFF = дешевле но дороже по ауре)
